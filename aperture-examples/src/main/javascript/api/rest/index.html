aperture.capture

Captures a snapshot of any web content as an image, for reporting or embedding in other live content. High performance throughput is enabled by a live pool of PhantomJS renderers, equipped as service providers.

aperture.layout

A layout service that includes two custom high-performance layouts for force-directed graphs and label / tag deconfliction, as well as common layouts backed by open source libraries such as JGraph, or yFiles for those with a commercial license.

aperture.palette.icon

The icon service is designed to serve ontological collections of symbols for semantic types. Vector source icons are dynamically served at any requested size in png, jpeg, or svg format. A base ontology of common analytic types is provided.

aperture.store

The Aperture store provides a simple, abstracted API to a Content Management Service (CMS), for storing and retrieving synchronized, versioned resources in a Service-Oriented Architecture. Abstracting CMS operations enables greater portability across deployment environments. A Derby based implementation is provided as well as a lightweight Ehcache version.

REST Web Services

Aperture services are implemented in REST. The JavaScript API wraps these service interfaces for JavaScript clients, adding a minimal amount of message formatting and handling for convenience. Other clients may make use of Aperture services by calling their REST interfaces directly.

Common MIME Types

Aperture REST services currently exchange all messages as application/json, except where services return images. Support for XML and / or binary protocol buffers are future considerations.

Optional, Modular Deployment

The client JavaScript library exposes Aperture services for optional use. It is possible to write Aperture visualizations that require no Aperture services at all. In these cases, deployment is simply a matter of including aperture.js.

Similarly, Aperture services can be deployed and used without the client JavaScript library, as independent modules. The easiest method of deploying the full collection of services is to simply drop a copy of the compiled aperture.war and server config file into a web server. However, clients may alternatively choose to package and deploy only those services required for a particular application.
